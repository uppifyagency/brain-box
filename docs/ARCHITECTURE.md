# Architecture & parameters

> For the developer who must **understand, operate and recreate** the system. Every value
> below comes from the production box (Oracle Cloud Free ARM, 2026) and latencies are
> MEASURED, not estimated. Companion: [DEPLOY.md](DEPLOY.md).

## 1. What it is, in one paragraph

A **self-hosted company brain**: a ~25 KB `box/` folder that, copied onto an Ubuntu VM
with Docker, becomes a complete system вАФ employees drag files into a Desktop folder вЖТ
files travel encrypted to the box вЖТ get converted to markdown, versioned in git, indexed
with vector embeddings вЖТ and become queryable in natural language from Claude Code /
Codex / Gemini via MCP. A local LLM (Gemma) extracts facts and synthesizes вАФ **no byte
leaves the box**. Engine: [gbrain](https://github.com/garrytan/gbrain) (MIT). The shell
(gate, ingest, client-kit, networking, ops) is this repo.

**Non-negotiable principles**
- `brain/<department>/` = markdown in git = **system of record**. Postgres is a derived
  cache, rebuildable with `gbrain sync --force`.
- One network door: **Tailscale** (private tailnet; public exposure is a separate,
  deliberate decision).
- The token is the perimeter: every access uses a per-employee bearer validated by gbrain.
- Provenance is server-side: who uploaded what is stamped by the server, never declared
  by the client.
- Per-instance secrets: `.env` is generated by `install.sh`, never in the template or git.

## 3. The flows (measured latencies)

### 3.1 DROP вАФ file вЖТ knowledge (~75s + ~1 min embedding)
1. Human drags a file into `Desktop/Company Brain/<dept>/`.
2. **launchd** fires instantly (`WatchPaths`) + 60s fallback tick. The watcher: lock вЖТ
   stability check (size still for 2s) вЖТ `POST /upload?dept=X` with bearer +
   `X-Filename` + `X-Employee`. Outcomes: 200 вЖТ `Caricati/` (done) ¬Ј 4xx вЖТ sidecar
   `.errore.txt` with the reason ¬Ј 5xx/offline вЖТ file stays put and retries (the file IS
   the queue). *Measured: dropвЖТupload 4вАУ6s.*
3. **upload-gate**: validates the bearer BY CALLING gbrain (single source of truth on
   tokens) вЖТ validates department вЖТ sanitizes filename вЖТ 200 MB cap вЖТ streams to a
   staging file + atomic rename вЖТ writes provenance sidecar `<file>.meta.json`.
4. **ingest** (event-driven): **A** conversion (matrix in ¬І4.5) with provenance
   frontmatter вЖТ **B** git commit per department repo вЖТ **C** `sync` вЖТ `embed --stale` вЖТ
   automatic fact extraction on new operational files (capped, best-effort) вЖТ **D**
   `doctor` (monitoring only, never a gate). *Measured: dropвЖТpage in DB 75s.*
5. Async embedding вЖТ semantically searchable. *~1 min at steady state.*

### 3.2 QUERY вАФ agent вЖТ answer (seconds)
MCP over HTTPS, stateless server, hybrid search (bge-m3 1024-d vectors + keyword FTS +
boosts). рЯФі **Query expansion is OFF by default** (`models.expansion none:off`): with the
upstream default, full-sentence questions in non-English languages returned EMPTY (the
LLM rewrite buried the right hit). A bearer's write access is floored to its FIRST
department: MCP writes on other departments' pages return "page not found" while reads
work вАФ that's the perimeter, not a bug.
вП± UX truth: fresh content is findable by keyword in ~1вАУ2 min, answers semantic questions
only after the end-of-cycle embed (a few minutes more).

### 3.3 LOCAL LLM вАФ Gemma inside the engine (async, never interactive)
Gateway вЖТ OpenAI-compatible API вЖТ Ollama. Production model: `gemma4:26b-a4b-it-qat`
(15 GB MoE). It is a *reasoning* model: **the F14 no-think patch is mandatory** (¬І4.3) вАФ
fact extraction 23вАУ72s warm with it, 3вАУ5 minutes without. Verified uses: `extract_facts`
(typed facts; auto-created entities), `think` (cited synthesis + gap analysis).
рЯФі Heavy reasoning NEVER via MCP: past ~5 min the client disconnects and **the work is
lost** (measured). Use the detached runner on the box (`ops/think.sh`).

### 3.4 BACKUP вАФ cron 03:00, restore DRILLED
`ops/backup.sh`: `pg_dump | gzip` + `tar brain/` + config вЖТ `backups/`, retention 14
(~75 MB/12s at full corpus). `ops/restore.sh`: scratch DB + RENAME. рЯФі The restore drill
found the shipped restore script BROKEN (DROP+CREATE in one `psql -c` = implicit
transaction that DROP refuses). Fixed with separate statements. **A backup you never
restored is a hope, not a backup.**

### 3.5 GUARDIAN вАФ deterministic self-healing (cron */15)
Watches what the engine cannot see (containers, disk, ollama, inbox, backups) and applies
idempotent, mechanical cures from the runbook. `GUARDIAN_MODE=observe` logs without
acting (mandatory first night), `act` cures. Non-curable anomalies become pages in a
NON-federated `ops` source вАФ invisible to user searches.

| Sensor | Threshold | Cure |
|---|---|---|
| container not running | immediately | `up -d` |
| disk | >85% | alert |
| backup | >26h | alert |
| dept inbox stuck | >30 min | restart ingest, max 1/2h then alert |
| embed backfill dead (pending>0, no progress, ollama IDLE) | 1 round (~15 min) | clear verified-dead locks + relaunch embed |
| zombie-grinding (pending>0, no progress, ollama BUSY) | streak 8 rounds = 2h | restart ollama + clear + relaunch |

Why 2h for the last one: the CLI embed path commits **per-whole-page** вАФ a big book means
70вАУ90 min of busy ollama with zero commits, a LEGITIMATE state that matches the signature.

## 4. Parameters that matter

### 4.1 `config.yml` вАФ the only file the admin edits
`azienda` (company name вЖТ Desktop folder name) ¬Ј `reparti` (departments = drop folders =
federated sources = gate validation) ¬Ј `expose` (`tailnet` | `public`) ¬Ј `llm` (`off` |
`byok`).

### 4.2 `.env` highlights (generated per-instance, never committed)
| Variable | Production value | Note |
|---|---|---|
| `GBRAIN_CHAT_MODEL` | `openai:gemma4:26b-a4b-it-qat` | рЯФі THE key that makes chat exist (env plane!) |
| `GBRAIN_CHAT_REASONING_EFFORT` | `none` | рЯФі F14 вАФ kills "thinking" on thinking-class models. Unset = patch dormant |
| `GBRAIN_EMBED_CONCURRENCY` | `1` | рЯФі F12 вАФ NEVER the upstream default 20 on CPU |
| `OLLAMA_MEM_LIMIT` / `OLLAMA_KEEP_ALIVE` / `OLLAMA_MAX_LOADED_MODELS` | `19g` / `-1` / `2` | anti-OOM + no cold loads (compose defaults) |
| `DELERA_WHISPER_MODEL` | `large-v3-turbo` | A/V transcription (faster-whisper, CPU int8) |
| `DELERA_OCR_MAX_PAGES` | `10` | OCR page cap per scanned PDF (truncation is declared in the output) |

### 4.3 Engine patches (in `engine/Dockerfile`, gbrain pinned)
| Patch | What | Why |
|---|---|---|
| **F3** | ollama recipe `default_dims 768вЖТ1024` | bge-m3 emits real 1024-d vectors; the CLI flag alone is whitelisted away |
| **op_checkpoints workaround** | drop a broken CHECK constraint | upstream bug: every sync after the first died |
| **F11** | embedding `max_batch_tokens: 8000` | unlimited batches on CPU exceed the runtime's ~300s fetch timeout; the batcher estimates in CHARS (~4/token) and code-dense chunks are ~2 chars/token вАФ hence 8k, not more |
| **F12** | `GBRAIN_EMBED_CONCURRENCY=1` | default 20 (tuned for cloud APIs) = 20 parallel pages on ONE ollama slot = 20√Ч latency = timeout storms |
| **F14** | env-gated fetch wrapper on the chat call-sites injecting `reasoning: {effort}` AND `reasoning_effort` | thinking-class models think by default on EVERY OpenAI-compat endpoint (2вАУ4√Ч generated tokens). `think: false` on /v1 is IGNORED by ollama; `PARAMETER think` doesn't exist; system prompts don't disable it (all tried). Only active when `GBRAIN_CHAT_REASONING_EFFORT` is set |

### 4.4 The two configuration planes (source of silent bugs)
- **FILE/ENV plane** (read at boot): `GBRAIN_CHAT_MODEL` decides whether chat *exists*.
  Without it, `extract_facts` returns `[]` silently.
- **DB plane** (`gbrain config set`): `models.default`, `facts.extraction_model`,
  `models.expansion`, `search.mode`.
- рЯФі **THE THREE-FLIP RULE**: changing the chat model = flip env `GBRAIN_CHAT_MODEL` +
  DB `models.default` + DB `facts.extraction_model`, then restart. (The extraction model
  is decided by the DB key вАФ a test with env=new but DB=old silently runs on the old
  model and cost the reference team 2 hours.)
- рЯФі After every ollama container recreate: **blocking, verified warm-up** of the chat
  model вАФ a big model's cold load (~300s) exceeds the runtime fetch timeout and the
  first extraction dies silently. And `VAR=x sudo docker ...` does NOT pass the env
  (sudo strips it) вАФ use explicit values after `-e`.

**OCR defenses (all measured):** glm-ocr TRUNCATES above ~150 dpi вЖТ downscale to 1800px;
it LOOPS on sparse pages вЖТ `temperature: 0` + `num_predict` cap + cut-at-first-echo.
**Anti-empty-shell gate:** quarantine only if extracted-format вИІ source >50 KB вИІ text
<200 chars (v1 quarantined legitimate 130-byte memos вАФ learn from it).

### 4.7 Ollama models (Neoverse-N1 4-core reference)
| Model | Size | Role | Measured speed |
|---|---|---|---|
| bge-m3 | 1.2 GB | 1024-d embeddings | ~2.4 s/chunk |

### 4.8 Network & security
Admin only via tailnet SSH (public 22 dropped via iptables, persisted). Exposure via
`tailscale serve` (tailnet-only). рЯФі Never `funnel 443` вАФ in current Tailscale versions
funnel is per-port and would publish the upload endpoint too. Tokens: created per
employee, department grants live in `permissions.source_id` (first element = write
floor). Revocation: `gbrain auth revoke <name>` вАФ immediate.

## 5. Troubleshooting вАФ bugs already paid for (do NOT rediscover them)

| Symptom | Cause | Cure |
|---|---|---|
| "Healthy" box but semantic search finds nothing | first embed failed вЖТ backfill in cooldown | install.sh already pulls bge-m3 BEFORE everything |
| `Sync PARTIAL вА¶ checkpoint_unavailable` from the 2nd sync on | upstream op_checkpoints bug | constraint dropped by install.sh |
| Embedding stuck, "another backfill is already running" | stale lock after a mid-backfill restart | delete the stale lock row (verify the holder is dead) |
| Embedding timeout storms, small pages ok big ones never | F11+F12 (unlimited batches + concurrency 20) | already patched in the template (8k + concurrency 1); bulk runs ALWAYS with `--catch-up` |
| Counter frozen 60вАУ90 min but ollama busy | CLI path commits per-whole-page; a huge page is HEALTHY | wait; truth is in ollama's tasks/min |
| Full-sentence questions return EMPTY (keyword phrases work) | query expansion via local LLM (upstream default) | `gbrain config set models.expansion none:off` + restart |
| OCR output truncated or looping | glm-ocr: truncates >150 dpi, loops on sparse pages | defenses already in convert-docs.py (downscale, temp 0, cut-at-first-echo) |
| extract_facts/think take MINUTES on a thinking-class model | the model reasons by default on every OpenAI-compat endpoint | F14: set `GBRAIN_CHAT_REASONING_EFFORT=none` |
| You change the chat model but behavior doesn't change | only the env was flipped | three-flip rule (¬І4.4) |
| First extraction after an ollama restart silently fails | cold load > fetch timeout | blocking verified warm-up after every recreate |
| VM dies hard during model experiments (exit 137, SSH killed) | OOM: two chat models resident, no swap, no mem_limit | guard-rails in the template: 8G swapfile + mem_limit + MAX_LOADED_MODELS=2; `ollama stop` residents before loading a new one |
| MCP `think`/heavy calls: timeout and lost work | >5 min = client disconnects, engine persists nothing | detached runner on the box (`ops/think.sh`) |
| MCP writes on another department's page: "page not found" | write floor = first department of the token | by design; use the drop folder / CLI for other departments |
| `doctor --json` core dump | runtime wart under pressure (worse off Ubuntu 24.04) | it's monitoring only (`\|\| true`); NEVER use as a gate |
| Ubuntu 26.04: doctor crashes, flaky embeddings | unstable bun runtime on that kernel | HARD requirement: Ubuntu 24.04 LTS |

## 6. Operating discipline (what made it reliable)

- After EVERY change to the box: run the acceptance suite (6 end-to-end tests, ~20s).
  The nightly cron run is the safety net, not an excuse to skip the sync check.
- First execution of ANY new automation: supervised / observe-mode. Always.
- Template in the repo вЖФ runtime on the box: aligned on every change, in BOTH directions.
- Bulk embedding: always CLI + `--catch-up`; never trust the page counter alone вАФ truth
  is ollama's tasks/min.
- Model swaps: golden-master baseline first, Parallel Change (expand вЖТ migrate вЖТ
  contract), disk cleanup LAST and only after 24вАУ48h green.
